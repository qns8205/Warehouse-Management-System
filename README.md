# 📦 WMS 스마트 재고 & 대여/반납 관리 시스템 (WMS Smart Inventory & Rental Management System)

구글 스프레드시트(Google Sheets)와 구글 앱스 스크립트(Google Apps Script, GAS)를 실시간 백엔드로 활용하여, 복잡한 데이터베이스 구축 없이 3D/2D 보관 구역 모니터링, 실시간 재고 실사 관리, 바코드/링크 연동, 불량 자재 로그 기록 및 자재 대여/반납 시스템을 한 번에 제공하는 통합 웹 애플리케이션입니다.

---

## 🚀 주요 기능 (Features)

1. **3D/2D 스마트 보관 구역 (Sectors & Layout Editor)**
   - 자재 창고의 구조를 2D/3D로 시각화하여 확인
   - 드래그 앤 드롭, 회전(Rotation), 크기 조절(Scale)을 통한 창고 레이아웃 실시간 편집 (스프레드시트 PropertiesService에 저장)
   - 특정 구역을 선택하여 해당 랙에 입고된 자재의 목록과 실시간 수량을 바로 조회

2. **실시간 재고 실사 관리 (Inventory Management)**
   - 구글 스프레드시트와 양방향 실시간 동기화
   - 품명, 규격, 담당자, 비고 및 구매 링크(스마트 칩) 연동 지원
   - **구글 드라이브 업로드 이미지** 실시간 링크 변환 및 렌더링 지원 (I열)

3. **간편 자재 대여 및 반납 신청 (Rental & Return System)**
   - 현장 작업자를 위한 대여/반납 전용 키오스크 화면 제공
   - 대여 시 실시간으로 본 재고 수량에서 자동 차감, 반납 시 재고 자동 복원
   - **대여로그** 시트에 대여자 이름, 일시, 수량, 메모가 자동으로 영구 기록

4. **실시간 불량 자재 로그 (Defect Logging)**
   - 파손, 오염, 기능 오작동 등의 불량 이슈 발생 시 원클릭 등록
   - **불량로그** 시트에 실시간으로 기록 및 동기화

5. **강력한 어드민 보안 체계 (Admin & Security)**
   - 스프레드시트 내 **Users** 시트를 활용해 실시간으로 관리자 ID 및 비밀번호 목록 관리
   - 어드민 전용 기능(구역 추가/편집, 자재 생성/수정/삭제, 로그 상세 조회) 제어

## 🛠️ 구글 시트 & Apps Script 3분 연동 방법 (Setup Guide)

이 프로젝트는 별도의 서버 구축 없이 **구글 스프레드시트**를 데이터베이스로 사용합니다.

1. **구글 스프레드시트 준비**
   - 구글 드라이브에서 새로운 스프레드시트를 생성합니다.
   - 첫 번째 시트의 이름을 `시트1`로 지정합니다. (혹은 원하는 이름으로 변경 후 Apps Script 코드 상단의 `INVENTORY_SHEET_NAME`을 매핑해 주세요)

2. **Apps Script 코드 붙여넣기**
   - 구글 시트 상단 메뉴에서 **확장 프로그램 → Apps Script**를 클릭합니다.
   - 기존의 `myFunction()` 코드를 모두 지우고, 이 프로젝트의 `AppsScript_Code.gs` 파일에 포함된 전체 코드를 복사하여 붙여넣습니다.

3. **웹 앱으로 배포**
   - Apps Script 편집기 우측 상단의 **배포 → 새 배포**를 누릅니다.
   - 다음과 같이 설정합니다:
     - **유형 선택**: `웹 앱` (톱니바퀴 아이콘 클릭)
     - **설명**: `WMS Backend v1`
     - **웹 앱 실행 사용자**: `나 (Me)`
     - **액세스할 수 있는 사람**: `모든 사람 (Anyone)` *(외부 API 호출을 위해 반드시 '모든 사람'으로 설정해야 합니다)*
   - **배포** 버튼을 누릅니다. (최초 배포 시 구글 계정 권한 승인 팝업이 뜨면 `고급 → 이동`을 눌러 승인 완료합니다)

4. **웹 앱 URL 입력**
   - 생성된 **웹 앱 URL (`https://script.google.com/macros/s/.../exec`)**을 복사합니다.
   - 웹 애플리케이션의 시작 화면(Landing Page) 또는 우측 상단의 **연동 설정(Setup)** 창에 복사한 웹 앱 URL을 붙여넣고 연결합니다.

## 👥 로컬 실행 및 다른 사람들과 공유하는 방법

이 프로그램은 **서버가 없는 클라이언트 기반의 웹 애플리케이션(SPA)**입니다. 따라서 아주 간단하게 다른 사람들과 공유할 수 있습니다.

### 1. 로컬에서 구동하기 (Local Run)
개발자 본인 컴퓨터나 사내 로컬망에서 실행할 때 사용하는 방법입니다.

```bash
# 1. 의존성 패키지 설치
npm install

# 2. 로컬 개발 서버 실행
npm run dev
```
- 터미널에 출력되는 `http://localhost:3000` 또는 `http://192.168.x.x:3000` 주소로 접속합니다.
- 동일한 와이파이/공유기(로컬 네트워크)에 연결된 다른 사용자는 컴퓨터의 **로컬 IP 주소(예: `http://192.168.0.10:3000`)**를 입력하면 본인의 컴퓨터에서 띄운 서버로 바로 접속하여 함께 테스트할 수 있습니다.

### 2. 인터넷을 통해 전 세계에 공유하기 (무료 배포)
동일한 로컬망이 아니더라도, 다른 사람들이 언제 어디서나 접속할 수 있도록 인터넷에 올리는 가장 쉽고 대표적인 방법들입니다. 이 앱은 정적 파일만 빌드하므로 **완전 무료**로 호스팅할 수 있습니다.

#### 방법 A: AI Studio의 공유 기능 활용하기
- 현재 사용 중인 Google AI Studio의 **Share (공유)** 버튼을 누르면, 다른 사람도 바로 접근 가능한 공개 URL이 생성됩니다. 가장 빠르고 별도의 설정이 필요 없는 방법입니다.

#### 방법 B: Netlify 또는 Vercel에 배포하기 (추천 - 1분 소요)
1. 코드를 빌드하여 정적 파일을 만듭니다:
   ```bash
   npm run build
   ```
2. 빌드가 완료되면 루트 폴더에 `dist` 폴더가 생성됩니다.
3. [Vercel](https://vercel.com) 또는 [Netlify](https://www.netlify.com)에 로그인합니다.
4. 가입 후 화면에 보이는 Drag & Drop 영역에 빌드된 `dist` 폴더를 그대로 끌어다 놓으면 몇 초 만에 전 세계에 공유 가능한 나만의 주소(`https://my-wms.vercel.app`)가 즉시 발급됩니다.

#### 방법 C: GitHub Pages로 배포하기
1. 이 프로젝트를 GitHub 저장소(Repository)에 업로드합니다.
2. 저장소 설정(Settings) -> Pages 메뉴에서 `Build and deployment` 소스를 `GitHub Actions` 또는 `main` 브랜치의 `/docs` 폴더(혹은 `gh-pages` 브랜치)로 지정하여 자동으로 배포할 수 있습니다.

## 📝 관리 및 유지보수 팁

- **사진 업로드**: 구글 드라이브에 올린 사진의 "링크 복사"를 통해 획득한 주소를 자재 사진 등록 칸에 붙여넣기만 하면, 시스템이 자동으로 이미지 URL 형태로 실시간 변환하여 렌더링합니다.
- **수량 동기화**: 대여 및 반납 신청이 완료되는 즉시, 구글 시트의 재고 열(`E열`)에서 자동으로 숫자가 갱신되므로 장부 누락을 완전히 방지할 수 있습니다.
- **어드민 계정 추가**: 구글 스프레드시트의 `Users` 탭에서 ID와 비밀번호 열에 행을 추가하면, 실시간으로 어드민 계정이 동기화되어 추가된 계정으로 즉시 로그인이 가능합니다.
